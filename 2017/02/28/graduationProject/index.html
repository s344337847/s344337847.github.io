毕业设计啊！——想法篇

发表于 2017-02-28   |  

前话

不知道什么鬼，感觉上班一个月就像过了两个月。

迷迷糊糊的又懒了一个月。

突然想起来还要个毕业设计 + 论文。

此时我的内心是

我..

我是正文

哎！老师我可以不交吗？

可以啊！如果你不想毕业的话!

于是这几天我苦思冥想，终于皇天负了有心人，我还是没想到做什么好！去他喵的大道理。

但是！

今天下班的路上我突然有了个想法，浪里个浪。

就做个漫画APP的吧。

现在开始整理下思路

首先

我已经知道我要做什么了！对吧一个APP！

第二我们就要考虑用什么技术去实现，这不是废话吗！

在学校学的也算是杂，网页，后台，Android都有学，但拿的出手的也就只有Adnroid，其它学的相对较少。那我们就用网页来实现吧！我就是喜欢不按套路出牌，浪里个浪。

其实现在在考虑的是如何解决Server端的问题

1、第三方后端云，方便。

2、自行实现，工作量加倍，但是大大增加灵活性。

我用过第三方后端云，有好自然也有坏，看你怎么取舍，毕业设计就不打算使用他了

如果是自行实现后台，首选自然是PHP,快速开发无法反驳。

可以没学PHP啊！啊！啊！

也就只能用Java了，没得选

数据库的话也没得选MySQL

嗯！要想想APP具体功能了

一个漫画APP，看漫画是主要功能，漫画就是图片，咱们客户端就是个图片浏览器，服务器端就是下载图片，没错，这个思路肯定不会掉坑里

其实也还有很多功能，登录注册、点赞收藏、漫画点评、实时弹幕、炫酷叼炸天的交互特效等等，反正一切从简，嗯，从简。

功能想出来后就要设计数据库，接下来开始忙活吧，懒了一个多月
